// Latihan 2: Prototype // Sebelumnya, kita udah membuat Object baru dengan menghubungkan methodSiswa sebagai metode-nya, jadi kita tidak perlu memasukkan metode-nya ke setiap siswa-nya, kita hanya perlu mengaitkannya dengan Object.create(). // Sekarang, kita akan membuat Object baru dengan cara lain yang orang bilang lebih efektif, kita menggunakan prototype // Prototype dalam JavaScript adalah cara untuk mewariskan metode atau properti ke objek lain. Prototype sangat berguna saat ingin membuat metode yang dapat dipakai oleh banyak object tanpa kita menduplikasi metode-nya ke masing-masing object. Sebenarnya Object.create() termasuk dalam Prototype itu, tapi Object.create() menentukan prototype-nya secara manual, nah sekarang kita akan mencoba cara lain dari prototype // Membuat metode-metode object, lalu dimasukkan ke prototype Constructor Function-nya Siswa.prototype.makan = function(porsi) { this.energi += porsi; } Siswa.prototype.tidur = function(jam) { this.energi += jam * 2 } Siswa.prototype.bermain = function(lama) { this.energi -= lama * 1.5 } function Siswa(nama,energi) { this.nama = nama; this.energi = energi; } let siswa1 = new Siswa('Diandra', 5); // Jadi metode makan, tidur, dan bermain itu dimasukkan ke dalam prototype-nya Siswa. Jadi ketika kita mengakses salah satu metode-nya dengan instansi-nya seperti siswa1, JavaScript awalnya tidak akan menemukan apa-pun didalam-nya, lalu setelah itu akan menelusuri melalui prototype chain ke pembuat Object-nya yaitu Siswa bagian prototype, nah ketemu lah metode-nya didalam prototype Siswa. // Metode yang kita gunakan disebut Prototype Inheritance. Sebenarnya ada cara selain menggunakan metode ini, yaitu menggunakan class. Class itu seperti template untuk membuat object baru class Siswa2 { constructor(nama,energi) { this.nama = nama; this.energi = energi; } makan(porsi) { this.energi += porsi * 1.2 } tidur(lama) { this.energi += lama * 2 } bermain(waktu) { this.energi -= waktu * 1.5 } } let siswa2 = new Siswa2('Mita',10) // Sebetulanya versi class, dibelakang layar menjalankan .prototype ini, jadi class hanya membungkus semua lebih masuk akal
